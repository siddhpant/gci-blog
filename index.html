Google Code-in 2018 Journey Hello there! I'm Siddh, and in this website I've somewhat elucidated my experience of GCI 2018.
The experience is listed task-wise, so open the different sections using the menu!
I worked for two organizations: coala and SCoRE.

I initially worked for SCoRE and did some tasks quickly as they were pretty beginner type.
Later I stopped working for SCoRE as the tasks were similar, and everyone was doing the same thing, and the tasks got approved.
Also, the mentors were not much active.

So I did tasks of coala and found it good! The community is active, the repos are actively maintained, mentors were helpful, and you had to do actual work.
Overall, my experience was pretty good!

coala
Ⓩ

coala provides a unified interface for linting and fixing code with a single configuration file, regardless of the programming languages used. Plugins may by native Python or wrap external linters.

coAST: Add a property to the languages schema

In this coAST task, I had to add a new property to the coAST models/schema in order to have more generality and abstraction.

I added a precision attribute, which is important for floating-point types. I earlier used this attribute without adding this in schema.

The change was reflected in my open PRs about C and C#.

username-availability: Issue #113: Detect real names

In this task, I had to enforce a check if the username provided is not a real name, as using full real names for usernames is not a good idea.

This was essentially a NLP task, so I had to lookup about NLTK which was new to me! So a new toolkit in my box. Made a python script

And then I had to integrate it with the project, so some JavaScript and Jinja2 trickery took place.

gh-board: Issue #114: Add link to PR /files tab

Someone was raving on Zulip about codebase while doing this task and then abandoned the competition itself, so I took it up and did it!

coAST: Add a new language definition *not* supported by coala

In this coAST task, I had to add a language definition which was not supported by coala. So I added x86 definition.

I initially added both uppercase and lowercase keywords (since both work), but then static broke due to too many keywords. :-P
